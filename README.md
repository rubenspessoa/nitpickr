# nitpickr

`nitpickr` is a self-hosted GitHub PR reviewer that reads the diff, applies repo instructions, posts inline review comments, publishes a summary, and includes a Mermaid flow graph in the review body.

## Current Status

This repository is moving from usable alpha toward a reliability-focused beta:

- GitHub App integration only
- Postgres-backed queue and review persistence
- API and worker processes
- Docker Compose for local/self-hosted deployment
- Repo instructions from `.nitpickr.yml`, `.nitpickr/`, and `AGENTS.md`
- Manual review trigger with `@nitpickr review`

## Quickstart

1. Install dependencies with `pnpm install`.
2. Generate starter config with `pnpm cli setup`.
3. Edit `.env` and fill in:
   - `OPENAI_API_KEY`
   - `GITHUB_APP_ID`
   - `GITHUB_BOT_LOGINS`
   - `GITHUB_PRIVATE_KEY`
   - `GITHUB_WEBHOOK_SECRET`
   - `NITPICKR_WEBHOOK_URL`
4. Leave `OPENAI_BASE_URL` and `GITHUB_API_BASE_URL` at their defaults for production, or point them at local stubs in test environments.
5. Start the stack with `docker compose up --build`.
6. Install the GitHub App on the repositories you want to review.
7. Open a pull request and comment `@nitpickr review`.

The default `.env` generated by `pnpm cli setup` is already wired for the bundled Compose Postgres service.

For local visibility, set `NITPICKR_LOG_LEVEL=debug` in `.env` and tail the services with `docker compose logs -f api worker`. The API logs startup, webhook handling, and failures. The worker logs job claims, planning, review publication, and job failures.

Run `pnpm cli doctor` before starting the stack. It now validates the core self-hosted setup, including the webhook path, GitHub private key format, and database URL shape.

## GitHub App Setup

Create a GitHub App and configure:

- Webhook URL: your public `https://.../webhooks/github` endpoint
- Webhook secret: the same value used in `GITHUB_WEBHOOK_SECRET`
- Repository permissions:
  - Pull requests: Read and write
  - Contents: Read-only
  - Issues: Read and write
  - Checks: Read and write
  - Metadata: Read-only
- Subscribe to webhook events:
  - Pull request
  - Issue comment

If you are running locally, expose the API with a public HTTPS tunnel such as `cloudflared` or `ngrok` and use that URL for `NITPICKR_WEBHOOK_URL`.

Manual review mentions are configured with `GITHUB_BOT_LOGINS`. By default the generated config accepts both `@nitpickr` and `@getnitpickr`.

## Beta Reliability Notes

- `nitpickr` publishes a `nitpickr / review` GitHub check run while a review is running.
- Review runs carry explicit states internally: running, published, failed, skipped, and superseded.
- Duplicate publishes are prevented with a stable per-run marker in the review body.
- Automatic reviews use a smaller inline comment budget than manual full reviews.
- Large PRs degrade to summary-only review instead of flooding comments.

For common failures, recovery steps, and the first-run checklist, see [docs/beta-runbook.md](/Users/rubenspessoa/Documents/Workspace/nitpickr/docs/beta-runbook.md).

## Current Limitations

- GitLab and Bitbucket are not implemented yet.
- Review memory is stored locally in Postgres and stays repo-scoped.
- The semantic context layer is diff-first today; AST integration is still deferred.
