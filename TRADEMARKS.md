## What you may do

- refer to this project by name when accurately describing or linking to the

- state truthfully that your product or fork is based on `nitpickr`

  software under the repository license

- use the `nitpickr` name, logo, or mascot in a way that suggests your fork,

- rebrand your own hosted or commercial offering as `nitpickr`
- remove attribution and present the project as if you created the original
  work
- use the mascot or other branding in marketing, product names, domains, or app
  listings in a way that is likely to confuse users about source or ownership

## Forks and derivatives

If you fork the project, you should rename the fork if there is any meaningful
chance users could mistake it for the upstream project or an official offering.

If you want to use the official `nitpickr` branding in a commercial context,
contact the maintainer privately first if you already have a private channel.
If you do not, open an issue requesting a private follow-up without posting
confidential commercial details, and the maintainer can move the conversation
to a private channel.
