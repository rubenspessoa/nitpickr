Review Quality

Benchmark

How we measure review quality across different models, and how you can reproduce the results yourself.

Why benchmark AI code review?

AI code review quality varies significantly across models, prompt strategies, and configuration. Without systematic measurement, it is difficult to know whether a model produces useful, accurate reviews or generates noise that slows down your team.

nitpickr includes a built-in evaluation framework (pnpm eval:reviews) that lets you benchmark review quality against real pull requests with known issues.

Methodology

The benchmark evaluates reviews along these dimensions:

  • Recall — does the reviewer find the known issues in the diff?
  • Precision — are the comments relevant, or does the reviewer generate false positives?
  • Actionability — are the comments clear enough for the author to act on without extra context?
  • Line accuracy — do inline comments point to the correct lines in the diff?
  • Summary quality — does the review summary accurately describe the change and key concerns?

Test corpus

The benchmark uses a set of real pull requests with human-annotated expected findings. Each PR has known bugs, style issues, or architectural concerns that a good reviewer should catch.

Scoring

Each dimension is scored on a 0–5 scale. The overall score is a weighted average that prioritizes recall and precision over formatting concerns. Results are reported per model and per prompt optimization mode.

Results

The following results reflect nitpickr's current review pipeline with default prompt optimization settings. Results will change as models are updated and the review pipeline evolves.

Model Recall Precision Actionability Line Accuracy Overall
gpt-5-mini 4.1 3.8 4.0 4.2 4.0
gpt-4.1 4.3 4.1 4.2 4.0 4.2
gpt-4.1-mini 3.6 3.5 3.7 3.9 3.6

Scores are on a 0–5 scale. Higher is better. Results are based on the current test corpus and may shift as the review pipeline and models are updated.

Key observations

  • gpt-5-mini delivers strong review quality at significantly lower cost than flagship models, making it a good default for most teams.
  • gpt-4.1 scores highest overall but costs more per review. Teams that need the deepest analysis may prefer it.
  • Smaller models like gpt-4.1-mini are usable for basic review but miss some subtle issues.
  • Repository-specific instructions (.nitpickr.yml) improve precision across all models by focusing the reviewer on what matters to your team.

Reproduce the benchmark

The evaluation framework is included in the nitpickr repository. You can run it yourself to validate results or test additional models. 

git clone https://github.com/rubenspessoa/nitpickr.git
cd nitpickr
corepack enable
pnpm install

# Set your environment
cp .env.example .env
# Fill in OPENAI_API_KEY and OPENAI_MODEL

# Run the evaluation
pnpm eval:reviews

The evaluation runner reports per-dimension scores, overall averages, and comparison tables when you test multiple models.

Limitations

  • The test corpus is small and focused on TypeScript and JavaScript. Results on other languages may differ.
  • LLM outputs are non-deterministic. Running the same benchmark twice may produce slightly different scores.
  • Scores reflect the current review pipeline. Improvements to prompting, chunking, and context retrieval will change results.
  • The benchmark evaluates the review pipeline end-to-end, not the raw model capability. nitpickr's prompt construction and output parsing contribute to the final quality.

Contributing to the benchmark

If you have pull requests with well-defined expected findings, contributions to the test corpus are welcome. See the contributing guide for details.
