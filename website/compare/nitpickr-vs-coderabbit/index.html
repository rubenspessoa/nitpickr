nitpickr vs CodeRabbit

How does a self-hosted, model-agnostic AI code reviewer compare to a SaaS platform? Here is a honest look at both approaches.

Overview

Both nitpickr and CodeRabbit automate code review on GitHub pull requests using large language models. The core difference is where the software runs and who controls the model.

Feature nitpickr CodeRabbit
Hosting Self-hosted (your infra) SaaS (their servers)
Model choice Any OpenAI-compatible model Vendor-selected models
Data privacy Code never leaves your infra Code sent to third-party servers
Pricing Pay only for LLM API usage + hosting Per-seat subscription
Source code Source-available (Elastic License 2.0) Proprietary, closed source
Configuration In-repo (.nitpickr.yml, AGENTS.md) Dashboard + in-repo config

Self-hosted vs SaaS

nitpickr runs on your own infrastructure. That means your source code, review logs, and model traffic stay within your operational boundary. You control the deployment, the secrets, and the runtime.

CodeRabbit is a SaaS product. You install it from the GitHub marketplace and reviews happen on their servers. This is faster to set up but means your code is processed on third-party infrastructure.

Model control

With nitpickr, you choose the model. Set OPENAI_MODEL and OPENAI_BASE_URL to use GPT-4, GPT-5-mini, Claude, Gemini, or any provider that exposes an OpenAI-compatible API. You can switch models at any time without changing your workflow.

CodeRabbit uses their own model selection. You generally cannot bring your own API keys or choose a specific model version.

Pricing

nitpickr has no subscription fee. You pay for your own LLM API usage and whatever hosting you choose. Running locally with Docker Compose and a free HTTPS tunnel (like cloudflared) means you mainly pay for OpenAI API calls.

CodeRabbit uses per-seat pricing with monthly or annual subscriptions. The free tier has usage limits.

When to choose nitpickr

  • You want your code to stay on your own infrastructure
  • You want to choose and switch AI models freely
  • You prefer paying only for actual API usage instead of per-seat fees
  • You value source-available software you can inspect and modify
  • You want review configuration to live in your repository, not a dashboard

When to choose CodeRabbit

  • You want a zero-setup experience with no infrastructure to manage
  • You need GitLab, Bitbucket, or Azure DevOps support today
  • You prefer a managed SaaS with a web dashboard
  • You do not need control over which model is used

Frequently asked questions

What is the main difference between nitpickr and CodeRabbit?

nitpickr is self-hosted and source-available, meaning you run it on your own infrastructure and bring your own API keys. CodeRabbit is a SaaS platform where your code is sent to their servers for review. With nitpickr, your code, logs, and model traffic never leave your infrastructure.

Can I choose which AI model nitpickr uses for code review?

Yes. nitpickr is model-agnostic. You set the model through the OPENAI_MODEL environment variable and can use any OpenAI-compatible API endpoint via OPENAI_BASE_URL. This means you can use GPT-4, GPT-5-mini, Claude, Gemini, or any other model that exposes an OpenAI-compatible API.

How does nitpickr pricing compare to CodeRabbit?

nitpickr has no per-seat or subscription fees. You pay only for your own LLM API usage and hosting costs. Running locally with Docker Compose and a free HTTPS tunnel means you mainly pay for model usage. CodeRabbit charges per-seat subscription fees on top of any underlying model costs.

Does nitpickr support GitHub, GitLab, and Bitbucket?

nitpickr currently supports GitHub through a GitHub App integration. GitLab and Bitbucket support are on the roadmap. The internal architecture is already provider-agnostic to support future expansion.
