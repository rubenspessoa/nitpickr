Quick Start Guide

Getting Started

Go from zero to your first AI code review in under ten minutes using Docker Compose.

Requirements

  • Node.js >=20.9.0
  • pnpm 10
  • Docker and Docker Compose
  • A GitHub account where you can create a GitHub App
  • An OpenAI API key
  • A public HTTPS URL for GitHub webhooks (use cloudflared or ngrok locally)

1. Clone and install

Clone the repository and install dependencies:

git clone https://github.com/rubenspessoa/nitpickr.git
cd nitpickr
corepack enable
pnpm install

2. Copy the environment template

cp .env.example .env

3. Create a GitHub App

Follow the GitHub App setup guide to create your app, set permissions, generate a private key, and install it on the repositories you want reviewed.

Required GitHub App permissions:

  • Pull requests: Read and write
  • Contents: Read-only
  • Issues: Read and write
  • Checks: Read and write
  • Metadata: Read-only

Subscribe to these webhook events:

  • Pull request
  • Issue comment

4. Start a public HTTPS tunnel

GitHub needs to reach your local machine. Start a tunnel that points to http://localhost:3000.

Important: use the tunnel origin for both NITPICKR_BASE_URL and NITPICKR_WEBHOOK_URL. Do not leave NITPICKR_BASE_URL at localhost.

5. Fill in your .env

The key variables you need to set:

DATABASE_URL=postgresql://nitpickr:nitpickr@localhost:5432/nitpickr
NITPICKR_BASE_URL=https://your-tunnel-origin.example.com
NITPICKR_WEBHOOK_URL=https://your-tunnel-origin.example.com/webhooks/github
NITPICKR_SECRET_KEY=<generate with openssl rand -hex 32>
OPENAI_API_KEY=<your OpenAI API key>
OPENAI_MODEL=gpt-5-mini
GITHUB_APP_ID=<numeric app ID>
GITHUB_PRIVATE_KEY=<PEM content>
GITHUB_WEBHOOK_SECRET=<your webhook secret>
GITHUB_BOT_LOGINS=nitpickr-dev

For local Docker Compose, the default DATABASE_URL already targets the bundled Postgres container.

6. Validate the setup

set -a
source .env
set +a
pnpm cli doctor

7. Start the local stack

The Docker Compose file runs three services: a Postgres database, the API, and the worker. 

docker compose up --build

8. Verify the service

curl http://localhost:3000/healthz
# Expected: {"ok":true}

curl http://localhost:3000/readyz
# Confirms database connectivity and worker heartbeat

9. Run your first review

Install the GitHub App on a repository, open a pull request, and comment: 

@nitpickr review

You should see a nitpickr / review status check appear, followed by inline comments and a review summary.

Review commands

Manual commands you can use in PR comments:

  • @nitpickr review — trigger a standard review
  • @nitpickr full review — deeper analysis
  • @nitpickr summary — summary only
  • @nitpickr recheck — re-run the review
  • @nitpickr ignore this — dismiss a finding

Reply commands for existing nitpickr comments:

  • @nitpickr why — explain the reasoning
  • @nitpickr teach — learn from your feedback
  • @nitpickr fix — suggest a fix
  • @nitpickr reconsider — re-evaluate

Repository configuration

nitpickr reads instructions from .nitpickr.yml, the .nitpickr/ directory, and AGENTS.md in your repository root. 

version: 1
review:
  strictness: balanced
  maxComments: 10
  focusAreas:
    - API correctness
    - background jobs
    - migrations
