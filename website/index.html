Model control

Keep the model and the rules under your control.

nitpickr is source-available under Elastic License 2.0. You can inspect the codebase, keep the review instructions next to the repository, and decide which model endpoint the reviewer should use.

  • OPENAI_MODEL defines the current review model.
  • OPENAI_BASE_URL lets you point at a compatible endpoint.
  • Prompt optimization modes help keep large PRs more economical.
  • The code and the configuration surface stay visible to your team.
runtime config 
DATABASE_URL=postgres://...
NITPICKR_BASE_URL=https://your-host
OPENAI_MODEL=gpt-5-mini
OPENAI_BASE_URL=https://api.openai.com/v1
NITPICKR_PROMPT_OPTIMIZATION_MODE=balanced

Practical default: start with gpt-5-mini, then move up only when the extra review depth is worth the spend.

FAQ

A few quick questions.

Is nitpickr open source?

The project should be described as source-available, not OSI-approved open source. The code is published under Elastic License 2.0, which allows personal and internal commercial use but restricts offering nitpickr itself as a hosted or managed service.

Why would I host this on my own VPS instead of using a hosted tool?

Because some teams want the runtime, logs, secrets, and model traffic under their own operational boundary. nitpickr keeps the architecture simple enough that a private VPS is a real option, not a marketing checkbox.

What can I control today?

The deployment location, the GitHub App installation scope, the repository instruction surface, the review strictness, and the model endpoint through OPENAI_MODEL and OPENAI_BASE_URL.

What does the product support right now?

Today nitpickr is GitHub App only. It reviews pull requests, publishes inline comments, writes a summary body, and updates a nitpickr / review status check.
